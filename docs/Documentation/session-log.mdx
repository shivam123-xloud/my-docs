## Structural Changes

<AccordionGroup>
  <Accordion title="RBAC: Split from 1 page to 5 pages">
    The original single `rbac.mdx` contained the overview, role hierarchy, configuration, and all 4 topic sections in one file. The 4 topics were extracted into dedicated pages registered under the RBAC nav group:

    - Resource Group Access
    - Data Protection Access
    - Service Plans Access
    - Script Execution & Console Access
  </Accordion>
  <Accordion title="Cluster Lifecycle Management: Split from 1 page to 4 pages">
    The original `cluster-lifecycle.mdx` contained expansion, shrink, and all recovery paths in one large file. Split into:

    - Cluster Lifecycle (overview + health + validation + best practices)
    - Capacity Expansion
    - Controlled Node Removal
    - Failure Recovery (evacuation + service restart + manual DB + UID/GID + power loss DR)
  </Accordion>
  <Accordion title="User Profile: Split from 1 page to 8 pages">
    The original `user-profile.mdx` contained all dashboard operations in a single 864-line file. Split into 8 focused pages:

    - User Profile (overview + dashboard access + account settings + best practices)
    - Profile & Account Security
    - Security Configuration
    - Image Management
    - Instance Management
    - Volume Management
    - Network Management
    - Object Storage
  </Accordion>
  <Accordion title="Gap Patches Applied">
    Four product requirement gaps were identified and patched into existing files:

    | Gap | File Patched | What Was Added |
    |---|---|---|
    | CIFS backup target | `volume-backups.mdx` | CIFS tab in Backup Driver Configuration with `cifs-utils` mount options |
    | Auto-protection of new VMs | `volume-backups.mdx` | Pool-level RBD schedule + cloud-init hook + Nova notification listener |
    | User profile (photo / 2FA / VM credentials) | `user-profile.mdx` → `profile-account-security.mdx` | Three full sections covering all three topics |
    | Resource grouping (Clouds, datastores, pools) | `resource-allocation.mdx` | Resource Groups section mapping all named resource types to Xloud constructs + user assignment model |
  </Accordion>
