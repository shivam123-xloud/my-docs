---
title: Access Corrupted VM Data
description: Recover data from a corrupted virtual machine using snapshot and volume attachment.
---

If a **Virtual Machine (VM)** becomes corrupted, its data can still be recovered.

You can:

1. Create a **snapshot** of the corrupted VM  
2. Create a **volume** from that snapshot  
3. **Attach** the volume to a working VM  
4. Mount the volume and retrieve your data  

---

Access the Xloud portal using the link below:

<Button href="https://portal-del.xloud.tech/auth/login/?next=/">

</Button>

## Step 2: Navigate to Instances

Follow the navigation path:

`Project → Compute → Instances`

<Steps>
  <Step>Go to the <b>Project</b> tab</Step>
  <Step>Open the <b>Compute</b> section</Step>
  <Step>Click on <b>Instances</b></Step>
</Steps>

<Note>
Make sure your account has <b>Project → Compute</b> permission to view instances.
</Note>

<Image
  src="../images/creating-vm/2.png"
  alt="Instances Page"
  width="400"
/>

---

## Step 3: Create a Snapshot of the Corrupted VM

<Steps>
  <Step>Locate the corrupted VM in the Instances list</Step>
  <Step>Click <b>Create Snapshot</b></Step>
  <Step>Enter a Snapshot Name</Step>
  <Step>Click <b>Create Snapshot</b> to begin</Step>
</Steps>

<Image
  src="../images/access-corrupted-vm/12.png"
  alt="Create Snapshot Option"
  width="400"
/>

<Image
  src="../images/access-corrupted-vm/13.png"
  alt="Snapshot Creation"
  width="400"
/>

<Warning>
Ensure the VM is shut down or properly quiesced before taking a snapshot to avoid filesystem corruption.
</Warning>

---

## Step 4: Create a Volume from the Snapshot

Navigate to:

`Project → Compute → Images`

<Image
  src="../images/access-corrupted-vm/14.png"
  alt="Images Section"
  width="400"
/>

<Steps>
  <Step>Locate the snapshot created earlier</Step>
  <Step>Click the arrow icon to open options</Step>
  <Step>Select <b>Create Volume</b></Step>
  <Step>Enter Volume Name</Step>
  <Step>Add Description (Optional)</Step>
  <Step>Click <b>Create Volume</b></Step>
</Steps>

<Image
  src="../images/access-corrupted-vm/15.png"
  alt="Snapshot List"
  width="400"
/>

<Image
  src="../images/access-corrupted-vm/16.png"
  alt="Create Volume Option"
  width="400"
/>

<Image
  src="../images/access-corrupted-vm/17.png"
  alt="Volume Creation Form"
  width="400"
/>

<Info>
The newly created volume now contains a copy of the corrupted VM's disk data.
</Info>

---

## Step 5: Attach the Volume to a Working VM

To access the data:

`Project → Compute → Instances`

<Steps>
  <Step>Select a healthy (running) VM</Step>
  <Step>Click <b>Attach Volume</b></Step>
  <Step>Select the newly created volume</Step>
  <Step>Log in to the VM via SSH</Step>
  <Step>Mount the attached volume</Step>
  <Step>Copy the required files</Step>
</Steps>

<Tip>
Mount the volume in read-only mode to prevent accidental modification of original data.
</Tip>
