## What is a Subnet?

A **Subnet** is a subdivision of an IP network range that allows you to define a smaller, manageable portion of IP addresses within a larger network.

**Key characteristics:**
- **IP address subdivision** - Divides larger network into smaller segments
- **Network management** - Provides organized and efficient IP address allocation
- **Resource isolation** - Enables logical separation of network resources
- **DHCP support** - Supports automatic IP address assignment and network configuration
- **Gateway routing** - Controls network traffic routing and access

## Step 5: Define Subnet Parameters

Go to the **Subnet** tab and specify the following subnet parameters:

<Steps>
  <Step><b>Subnet Name</b>: Specify a descriptive name for the subnet</Step>
  <Step><b>Network Address</b>: Specify the IP address for the subnet in CIDR notation (e.g., 10.0.0.0/24 where the prefix is the binary representation of the network address and the suffix declares the total number of bits in the entire address)</Step>
  <Step><b>IP Version</b>: Select <b>IPv4</b> for Internet Protocol version 4</Step>
  <Step><b>Gateway IP</b>: A gateway IP refers to a device on a network which sends local network traffic to other networks. To use the default gateway, leave this field blank
  ![65](/Documentation/images/network-management/65.png)</Step>
</Steps>

<details>
<summary><b>ðŸ“‹ CIDR Notation Examples - Click to Expand</b></summary>

**Common CIDR Ranges:**
- **10.0.0.0/24** - 254 usable addresses (small networks, development)
- **10.0.0.0/22** - 1022 usable addresses (medium networks, production)
- **10.0.0.0/20** - 4094 usable addresses (large networks, enterprise)
- **10.0.0.0/16** - 65534 usable addresses (very large networks)

</details>

<Tip>
Plan your CIDR notation carefully during the design phase to ensure you have enough IP addresses for current needs and future growth.
</Tip>

## Step 6: Configure Subnet Details

Switch to the **Subnet Details** tab to configure additional settings:

<Steps>
  <Step><b>Enable DHCP</b>: Select this check box to enable DHCP</Step>
  <Step><b>Allocation Pool</b>: Define the range of IP addresses to be assigned by DHCP (optional, uses default range if not specified)</Step>
  <Step><b>DNS Name Servers</b>: Specify the DNS server names for name resolution. Common options include: Google DNS (8.8.8.8 8.8.4.4), Airtel DNS (202.56.230.2 202.56.230.7), or Cloudflare DNS (1.1.1.1 1.0.0.1)
  ![66](/Documentation/images/network-management/66.png)</Step>
</Steps>

<Info>
DHCP (Dynamic Host Configuration Protocol) automatically assigns IP addresses to devices on the subnet, making network configuration easier and more efficient without manual IP assignment.
</Info>

## Subnet Configuration Best Practices

**Network Planning Considerations:**
- **CIDR notation** - Choose appropriate subnet mask for your resource needs
- **IP addressing** - Plan for current usage plus future growth
- **Gateway configuration** - Use default or custom gateway based on network requirements
- **DNS resolution** - Select reliable DNS servers for name resolution
- **DHCP settings** - Configure appropriate allocation pools to avoid address conflicts

**Common Subnet Scenarios:**

*Development Environment:* Network 10.0.0.0/24 with 254 usable IP addresses, DHCP enabled for development and testing.

*Production Environment:* Network 10.0.0.0/22 with 1022 usable IP addresses, DHCP enabled with managed pools for business-critical applications.

*Large Enterprise:* Network 10.0.0.0/16 with 65534 usable IP addresses, DHCP enabled with segmented pools for multi-department deployments.

<Warning>
Plan your subnetting strategy in advance. Hasty or incorrect subnet configuration can lead to connectivity issues and operational challenges.
</Warning>
